Configuration - Use of Authorisation Group

Mangesh Nirmale

With help of authorisation group while creating DIR in DMS we can ristrict user for change/display for each document type. It is controlled through role assignment.
Some of the Authorisation Object are listed below.

Sr. No.Authorisation ObjectDescription
1 C_DRAW_BGRAuthorization for authorization groups
2 C_DRAW_DOKAuthorization for document access
3 C_DRAW_STAAuthorization for document status
4 C_DRAW_TCDAuthorization for document activities
5 C_DRAW_TCSStatus-Dependent Authorizations for Documents
5 C_DRAW_MUPAuthorization for Markups

 

Following steps are shown for one example. In DC10, for Document type DWG you have set, Authorization Group as Required Field (Double click on DWG document type in DC10, you can see your configuration for details of your configuration)

For Example....There are two users User A & User B and they both have access to document type XYZ. but they dont want to access each other document. So in this case we create two authorisation group as "AAA" for User A & "BBB" for user B. when they use their Authorisation group while creating DIR then they can access only those DIR which authoristion group matches with their respective role.

Step 1
T-Code : PFCG
Create a new role and goto Authorisation tab. Click change authorisation data

 

Step 2
Add object C_DRAW_BGR & C_DRAW_DOK.

 

Step 3
Under Document management enter the Authorization code of your choice, Document type, allowed activities
The user must remember this authorization code, as there is no F4 help for authorization group field in CV01N

 

Step 4
Click on generate profile and save.
Assign the newly created role to user
