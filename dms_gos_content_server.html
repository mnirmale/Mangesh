Configuration - GOS Attachment to be save in external HTTP server

Mangesh Nirmale
  • Word file of artical
    •
 

The diagram above shows relation of SOFFPHIO class and the storage Options for SAP Database or External HTTP content server.

By default the storage is defaulted to the SAP Database
Default path is SOFFPHIO-> SOFFDB->SOFFDB with this gos attachment will be save on sap database
To save GOS attachment to external contente server we need to change storage category to SOFFHTTP
change to this SOFFPHIO->SOFFHTTP->(*Custom Content Reposity*)

Step 1 : Define Content repository

T Code - OAC0 (SPRO->SAP Netwever->nowledge Management->Settings for knowledge warehouse system->Content management system->define content repository)
Create Content Repository (e.g Z_GOS). enter the server details



click 'Maintaine Content Repository' OR transaction CSADMIN
Type name of repository and click create
choose certificate tab and click send certificate
choose certificate line and click activate
Test the connection
Run RSCMST in se38 to chk content repository connection

Step 2 : Assign category SOFFHTTP to category SOFFPHIO

T Code - SKPR08
check in Transaction SKPR08 If there is entry of SOFFPHIO If exist put New Category as SOFFHTTP.
If class SOFFPHIO is not visible in above transaction then we have to execute following steps.
Step 2.1 :
Go to Tcode "SE16N" and enter Table "SDOKPHCL"
Step 2.2 :
Find Class "SOFFPHIO", select and click on EDIT. If not able to edit table then go to Function Module 'SE16N_INTERFACE' put table name in I_TAB, put X in I_EDIT
Step 2.3 :
Change Maint. Of Cat. To "X" and Repeat step 2

Step 3 : Assign category SOFFHTTP to content repository

T Code - OACT
Select entry SOFFHTTP.
In content repository enter your created content repo.

Step 4 : configuration for allowed file type

T Code - SOPE(Tools->Busines Communiation->Office->Filders->Document classes->SOPE)
Check required file type is not ristricted and add extra entries as needed

Step 5 : configuration gos menu

T Code - SGOS
Here you can enable or disable items for Generic Object Service (GOS) menu

Step 6 : configuration for FIORI

T Code - (SPRO->Cross Application Component->additional settings-simplification->Attachment service - storage repository activation)

Add new Entry for storage category "SOFFPHIO" & enable Active Indicator.

Step 7 : configuration for FIORI

T Code - (SPRO->Cross Application Component->additional settings-simplification->Attachment service - object type mapping)

Add new Entry for SAP Object & Object Type.
